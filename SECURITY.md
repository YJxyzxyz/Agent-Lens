# 安全策略

## 数据安全说明

AgentLens 作为 AI Agent 的追踪工具，**默认会将追踪数据保存在本地**。请注意以下安全事项：

### 敏感数据风险

追踪事件可能包含：
- LLM 的输入和输出内容
- 工具调用的参数和返回值
- 文件读取/写入的路径和内容
- 浏览器操作的目标 URL
- API key 或 token（如果代码中硬编码且被记录）

### 数据存储位置

所有 trace 数据默认保存在当前工作目录的 `.agentlens/runs/` 下，以 JSONL 格式存储。

### 安全建议

1. **不要提交 trace 文件**：`.agentlens/` 目录已在 `.gitignore` 中排除，请确保不要手动 `git add` 该目录
2. **不要在 issue 中贴原始 trace**：trace 数据可能包含敏感信息，报告 bug 时请脱敏
3. **审查记录内容**：在使用 `record_*` 函数时，注意不要记录 API key、密码等机密信息
4. **生产环境**：建议设置自定义存储路径到安全位置，并定期清理旧 trace

### 未来计划

- [ ] 自动 secret redaction（自动脱敏 API key 等）
- [ ] 可配置的敏感字段过滤
- [ ] 数据加密存储选项

## 报告安全漏洞

如果你发现安全漏洞，请**不要**在公开 issue 中报告。

请通过 GitHub Security Advisories 私密报告，或发送邮件至项目维护者。

我们会在 48 小时内确认收到报告，并在修复后公开致谢（如果你希望）。
